# SPEC: 第3世界「紅蓮の火山郷」（F3）

策定: 2026-07 / 担当: Fable(本書)→Opus(実装=O7)→Sonnet(シミュ/QA)→Codex(アート)
正史: STORY.md第3章 / CHARACTERS.md③ / GAME_STRUCTURE.md(エースLv24・記章③=烈火) / DIALOGUE.md③
物語ビート: **灰の使徒が火山の錬成炉を狙う → 老鍛冶イグナートと協力し撃退 → 「灰化は自然現象ではなく人為」と判明**（物語の転換点。ここから「誰が?」の謎が主軸になる）

## 0. 設計原則（過去の実測から）

1. **talent過剰ボスを繰り返さない**: シルヴァ初版(ACE talent6)は初見勝率0.4%=実質詰みだった(§8シミュで検出、talent1へ修正)。本世界のロードアウトは控えめに始め、**出荷前にシミュで50-60%を確認するまで数値は仮**とする。※未シミュのマレア(marinel Lv21 talent6)も同じ轍の疑いがあり、本世界のシミュ時に併せて検証すること。
2. **スターター公平性を地形設計に織り込む**: 消耗シミュ(S3)で「森は火スターターだけwipe率85%(他は35-40%)」という構造的不公平が判明。火山は素直に作ると風(コグリフ系)が同じ目に遭う(火→風2.0)。対策を§3に内蔵する。
3. **新システムは作らない**: パッケージDで実装済みの zones/ヌシ/スイッチ→ゲート/レッジ と、既存の warp.gate(記章ゲート)/TrainerData だけで組む。唯一の小拡張は`NushiSpot.lines?`(§5)。

## 1. 世界構成（海の世界と同型の2マップ）

```
WORLDS追加: { id:'volcano', name:'紅蓮の火山郷', icon:'🌋', mapId:'volcano_road',
              tx:<入口>, ty:<入口>, boss:'gym_fire', unlock:'蒼潮の記章',
              desc:'灼熱の溶岩回廊の先、鍛冶の郷。守護者イグナートの炉が赤く燃える。' }
```

- **volcano_road「溶岩回廊」**(28x30想定・biome:'volcano'): 黒曜石の岩場を縫う一本道+分岐のダンジョン。西端=転送門帰還、北東端=火山郷へ。
- **volcano_town「紅蓮の火山郷」**(24x16想定・biome:'volcano_town'、BGMはtown系): 鍛冶場が並ぶ町。守護者イグナート(ジム=大鍛冶場)、道具屋、宿、**錬成炉**(ストーリーイベント§5)。
- 進行ゲート(GAME_STRUCTURE「③烈火=溶岩路を通行」): volcano_roadの**北の脇道**に`warp.gate:'烈火の記章'`の出口を置く(第4世界・嵐嶺への接続予約。行き先は当面ダミーで「崩れた道だ」表示でも可)。

## 2. 溶岩回廊のマップ設計

- **新タイル 'M'=マグマ(進入不可)**: isWallに追加。CSSフォールバックは`field-zones.css`(赤橙の明滅グラデ)。`public/tiles/magma.png`で差し替え可。水路の'W'と同じ扱いで、描画だけ溶岩。
- 構造: サーペンタイン改(森の反省=完全一本道は単調)。**マグマ湖を島伝いに渡る中盤**+**レッジ('L')で降りる近道**(パッケージC資産の再利用)+**ヌシが塞ぐ宝箱ルート**+**スイッチ→ゲートの帰り道短絡**。
- ゾーン(§3)は5区画。森(6区画)よりひと回り小さくし、1世界の通過時間を先細りさせない(中盤マップは「森より短いが1戦が重い」リズム)。

## 3. エンカウント設計（スターター公平性を内蔵）

到達想定Lv: マレア戦(エースLv21)クリア直後の**Lv21-23**。

| 区画 | Lv帯 | プール方針 |
|---|---|---|
| Z1(入口) | 18-21 | 火stage1中心+**地stage1を2種混ぜる**(風の獲物) |
| Z2 | 20-23 | 火stage1/2+地 |
| Z3(マグマ湖) | 21-24 | 火/冥(candle系)+地 |
| Z4 | 22-25 | 火stage2中心 |
| Z5(最奥・郷の手前) | 23-26 | 火stage2+**レア枠15%=火stage3(volcadon等)** |

- **公平性の仕掛け**(原則2の実装):
  - 全ゾーンに**地タイプを25-35%**混ぜる。風→地2.0なので、コグリフ系は「火に殴られると痛いが、地相手にはエース」という往復のある体験になる(森のイグニフのような一方的な不利を作らない)。
  - **Z1に「湯けむりの岩場」**(温泉フレーバーの小部屋)を置き、**水タイプ野生(カッパル系等)を出現**させる。火山で詰みかけた風/火スターターが**その場で対策要員を捕獲できる**救済ルート。看板で「熱には水だ」とヒントも出す。
  - 検証基準(§7)に**スターター別wipe率の差≤15pt**を新設(S3の再発防止指標)。
- プール素材(canon優先): 火=emberio/candle系+生成(サラマ/マグマ/イフリッコ等9種)、地=森の生成地タイプから流用可、水=カッパル等。正確なid列挙は実装時に`wildOfTypes(['火','地'],…)`で。

## 4. ヌシ・スイッチ・宝箱（消耗設計込み）

- **ヌシ: ブレイズロア(blazeroar) Lv26 talent6**(★★確定)。Z4の分岐の奥、宝箱部屋を封鎖。守護者エース(イグニス・レオ)の**進化前**という設定で「炉で鍛えられ損ねた荒くれ」——倒せば守護者戦の予行、捕獲すれば育てて同系エースをぶつける楽しみ(イグナートが反応する台詞を§6に用意)。
- **スイッチ→ゲート**: ヌシの間の奥のルーンで、Z2⇔Z5を直結する帰り道を解放(森と同じ文法。プレイヤーに「この世界のお約束」として学習させる)。
- **宝箱(消耗設計)**: S3の実測「傷薬3個では足りない」を反映し、**回廊内に上傷薬×2箇所+傷薬×3+ゲル+フラスコの計5箱**。うち上傷薬1箱はZ3(マグマ湖)の見えるが遠い島=レッジ経由でしか取れない配置(寄り道リスク/リワード)。
- 町に道具屋があるため「枯渇したら郷まで走る」が成立する(森には無かった安全弁)。

## 5. ストーリーイベント「錬成炉の防衛」（既存機構だけで構成）

町の錬成炉前に**灰化幻獣(ヘルフレア Lv25 talent0・「灰化」状態で開始)**をヌシとして配置。
- **小拡張(Opus)**: `NushiSpot.lines?: string[]` を追加し、接触時の前口上を個別化できるようにする(現状は全ヌシ共通文言)。1フィールド+App.tsxのonNushi分岐1行で済む。
- 流れ: 町到着→イグナートに話しかけると「炉が狙われとる」→炉前の灰化ヘルフレアと強制戦闘(撃破/捕獲で解放)→イグナート「灰の使徒の仕業だ。**灰化は天災じゃねえ、誰かの仕業だ**」→ジム挑戦解禁(leaderへの接触が有効になる…は実装が重いので、**単に物語順の推奨**とし強制はしない。灰化幻獣を倒さずジムに勝った場合も台詞は破綻しない書き分けを§6でする)。
- シネル(灰の使徒幹部・portraits/シネル取込済)は**姿を見せず**、炉に残された「灰の紋章」の立て札だけで存在を示す(出し惜しみ。初対面は第7章の予定を守る)。

## 6. 守護者イグナート（gym_fire）

- 人物: 筋骨隆々の老鍛冶。頑固一徹・職人気質。口調「儂／〜い」。**鍛えた者を認める**——テーマ(時間をかけて育てる)の体現者側の人。
- ジム=大鍛冶場(volcano_town内、leaderをマップに配置)。バッジ=**烈火の記章**。

### ロードアウト（仮値。§7のシミュ通過まで確定させない）

| 枠 | 種 | Lv | talent | もちもの | 技 | 意図 |
|---|---|---|---|---|---|---|
| 1 | emberio(エンバーリオ) | 22 | 0 | - | 既定 | 露払い |

- ボスギミック: 開幕とうき(atk+2)→フレイムダッシュ(威力70・反動0.25)連打。**反動で自傷する**ので、耐えて守りを固める(まもりがため/みずのまく)か水技で畳み掛けるかの駆け引き。§8シミュの「火auraは主力と噛み合わない」問題は、**イグニス・レオが物理型(ATK122)なのでとうきが正しく機能する**——タイプでなく個体で噛み合わせる好例として設計。
- 難易度目標: **プレイヤーLv23・道具2個持込で初見勝率50-60%**(シルヴァと同基準)。調整ノブ: ACEのtalent(1→0)/Lv(24→23)/powerband外し/2枠目のLv。

## 7. 検証基準（Sonnet・出荷ゲート）

1. **守護者シミュ**: `scripts/sim_balance.mjs`を**トレーナーid引数対応に一般化**し(現状gym_forest直書き)、gym_fireを初見勝率50-60%に収める。**同時にgym_port(マレア)も測定**し、詰みなら§0-1の通りtalent/Lvを調整。
2. **消耗シミュ**: `scripts/sim_dungeon_resource.mjs`を火山ゾーン対応にし、(a)wipe率≤15%(標準探索・道具は宝箱込み) (b)**スターター別wipe率の差≤15pt** (c)郷到着時HP%が60-80%帯。未達なら§3のプール比率/宝箱/Lv帯で再調整(エンカ率は触らない——S3の実測でレベリング機会とのトレードオフが確認済み)。
3. **QA回帰**: `docs/QA_CHECKLIST.md`に「第3世界」行を追加(転送門解放条件→回廊→湯けむり捕獲→ヌシ→炉イベント→ジム→記章→北のゲート表示)。
4. 到達性: 有向BFS(ヌシ/ゲート/レッジ/記章ゲートの解決前後)で全宝箱・NPC・warpの到達を機械検証(パッケージC/Dと同じ手順)。
