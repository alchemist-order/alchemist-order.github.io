# 90日ロードマップ＆モデル分業体制（2026-07策定）

Fable(戦略/設計)・Opus(実装)・Sonnet(量産/定型)・Codex(画像/描画)の分業でPhase 1〜3を実行する。
各セッション開始時は本書＋プロジェクトメモリを読むこと。**メモリ更新はOpusが一元管理。**

## 勝ち筋（一文）
「タイムラインから3秒で始まる、大人のためのDQM。週替わりの塔で全員が同じ挑戦を競い、レア個体と隠し配合を自慢し合うゲーム」
— 本編は3〜4世界で「1.0」と宣言し、塔・錬成・レア収集を無限のリプレイ装置にする（コンテンツ踏み車を回避）。

## 役割定義
- **Fable** = プロデューサー/設計。判断が重いものだけ。**成果物は必ず docs/SPEC_*.md に残す**（Opusへの引き継ぎ媒体）
- **Opus** = リード実装。仕様書→実装→実機検証→コミット→デプロイまで一気通貫。Codex連携・メモリ管理
- **Sonnet** = ランブック駆動の量産/定型（アセットパイプライン・QA回帰・シミュ・表整備）
- **Codex** = 画像生成＋描画系コード（Field.tsx/index.css/素材）。従来分業を維持

## マネタイズ原則（要旨）
earned-first。能力値/幻獣/talentは売らない。コスメ＋シーズン記録帳(中身コスメのみ)＋支援者パック。
有料ガチャは導入しない。錬成レシピに課金要素を接続しない(コンプガチャ該当回避を構造的に保証)。
本命収益はSteam買い切りの可能性を温存。
