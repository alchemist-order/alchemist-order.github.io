# SPEC: 戦闘V3 「観るバトル、選ぶ捕獲」

策定: 2026-07 Fable（ブラウザ/カジュアルゲームの戦闘・捕獲リサーチに基づく）
実装: Opus/Codex（O14・最優先） / 前提: クイック決着(quickResolve即時モーダル)は本設計で**置換**する

## 0. 診断 — なぜ「クイック決着」は違和感があるか

操作コストと儀式性は別の軸である。

|  | 操作コスト | 儀式性(高揚の源) |
|---|---|---|
| 旧・手動戦闘 | 高(毎ターン選択) | 高(FX・捕獲の揺れ) |
| クイック決着 | ゼロ | **ゼロ(テキスト1枚)** |
| **V3(本設計)** | **ほぼゼロ** | **高(むしろ強化)** |

クイック決着は操作と一緒に「遭遇の顔が見える」「弱らせる過程」「フラスコの揺れ」「捕まえた！の瞬間」まで全部削った。
**削るべきは毎ターンのコマンド選択だけ**で、捕獲の儀式は1秒も削ってはいけなかった。

## 1. リサーチ結果（何が効いているか）

| 参照 | 戦闘 | 収集の喜び | 学び |
|---|---|---|---|
| Pokéclicker / 放置系ポケモンライク | 完全オート/クリック | 図鑑・色違いが主動機。捕獲自体は自動 | 戦闘ゼロでも収集欲だけでゲームが回る=**収集が本体、戦闘は舞台装置** |
| PokéRogue | フル手動維持 | ラン中の捕獲がビルドに直結 | 手動フルバトルは「高い緊張の場」でこそ効く=**ボス戦向き** |
| PokéFarm Q | 戦闘なし | 同種を連鎖させると色違い率UP(チェーン) | **「狙って粘る」導線が収集を沼化させる** |
| 放置少女/AFK系 | オート+倍速+スキップ。ただし**戦闘は見える**(キャラが動き数字が飛ぶ) | 編成=事前の意思決定 | オートでも「観て気持ちいい」演出は残す。**操作を消しても画面は消さない** |
| ドット伝説 等ドット放置 | わちゃわちゃ動くドットが売り | — | ドット絵が動くこと自体が報酬。AOのスプライト300体はここで活きる |
| ポケモン本家(捕獲処理) | — | ボール3回の揺れ=1回ごとの抽選演出、弱らせてから投げる駆け引き | **揺れ=分割された抽選こそドキドキの正体**。省略禁止 |
| ガチャの確定演出文法 | — | 演出の色や挙動でレア度を予告 | 捕獲の揺れに「金色/虹色」の予告を仕込める=変異種・高才能の開封体験 |

結論: **雑魚戦=観るもの(オート)、捕獲=選ぶもの(手動)、ボス戦=考えるもの(フル手動)** の3層に分ける。

## 2. 設計 — 3層バトル

### 2.1 雑魚戦: オート観戦バトル（探索の既定）
- 既存Battle.tsxを**自動操縦**する。技選択はAI(敵AI `chooseEnemyMove` の流用)、ターンは自動送り
- **画面は今のまま**: スプライト・ダメージポップ・弱点FX・SEすべて維持（放置少女の学び=操作を消しても画面は消さない）
- 速度: 既定2倍、設定で1x/2x/4x。1戦闘の体感 3〜8秒
- プレイヤーの腕の見せ所は**事前**に移る: 先頭を誰にするか(タイプ相性)・もちもの・ノード選択。「探索前の編成」が実質のコマンド入力

### 2.2 捕獲チャンス割込み（本設計の心臓）
オート進行中、以下の瞬間に**自動ポーズ**して選択パネルを出す:
- 敵HPが35%以下になった時（未捕獲種・変異種・高才能個体は50%で早めに割込み）

```
┌─ 捕獲チャンス！ ────────────────┐
│ 🔮 フラスコを投げる   成功率 62%      │←残HP/レア/お守り/状態異常から算出し明示
│ ⚔ さらに弱らせる     成功率UP・倒すリスク │←もう1ターンオート(やりすぎると倒して消滅)
│ 💨 とどめを刺す      経験値・ゲルへ       │
└─────────────────────────┘
```
- これが**1戦闘に1回の意思決定**=「欲をかく瞬間」。本家の「弱らせてから投げる」駆け引きを1画面に圧縮
- 投げたら**捕獲演出はフルで再生**(吸い込み→揺れ1〜3回→カチッ)。既存実装をそのまま使う
- **開封演出の追加**: 揺れの光を才能6+=金色、変異種=虹色に(確定演出文法)。「揺れの色で心拍が上がる」を作る
- 図鑑未登録種との遭遇時は開幕に**「NEW!」バッジ**表示（Pokéclickerの学び=「これは新種だ」が見えた瞬間に捕獲動機が点火する）

### 2.3 ボス戦: フル手動維持（守護者・ヌシ・試練の塔）
- 従来のコマンドバトルをそのまま。**「考えた瞬間」はここが担保**(3問テストQ1の置き場)
- ボス戦にも「オートに切替」ボタンは置くが既定は手動

### 2.4 敗北・撤退
- オート中に自分のHPが30%を切ったら自動ポーズ →[撤退する / 戦い続ける]（探索の深度リスクと接続）
- 撤退=そのイベントを失うだけ。全滅=探索戦利品半減(SPEC_NODE_EXPLORATION §3)

## 3. 実装ガイド（重要: ほぼ新規コード不要）

**quickResolve.tsの独自計算エンジンは廃止**し、Battle.tsxの自動操縦で置換する。理由:
1. FX・音・捕獲演出の資産をそのまま再利用できる
2. **戦闘計算が一本化**される（クイックと手動で勝率が違う二重経済バグの温床を根絶。S3のシミュ結果も引き続き有効）
3. 実装規模: Battle.tsxに `auto` モード(技選択AI+自動ターン送り+割込み条件)を足すだけ

```
src/screens/Battle.tsx :
  - autoMode state(探索雑魚戦=true / ボス=false初期値)
  - useAutoPilot(): 手番が来たら chooseEnemyMove 相当のヒューリスティックで技を選び doMove を呼ぶ
  - 割込み条件: enemy.hp <= threshold(未捕獲50%/通常35%) で pause → 捕獲パネル
  -             player.hp <= 30% で pause → 撤退パネル
  - 速度: 既存 battleSpeed を 1/2/4x に拡張
src/screens/Explore.tsx :
  - 「クイック決着」ボタン → 「オートで戦う(既定)」/「手動で戦う」に差し替え。即時モーダルは削除
src/game/quickResolve.ts : 廃止(将来の派遣調査のオフライン解決用に温存するなら export だけ残す)
```

- 捕獲率式は既存 `catchChance(enemy)` を使用し、パネルに%を明示。catch_charm補正もここに一本化
- 計測: `auto_battle{result}` `capture_panel{action, rate}` `capture_result{success, talent, mutant}`(O1)

## 5. 受け入れ基準

1. 探索での雑魚戦が**無操作で完走**し、体感3〜8秒(2倍速時)
2. 捕獲チャンスの割込み→%表示→投げる→揺れ演出→結果、が全て動く（演出スキップされないこと）
3. ボス戦(守護者/ヌシ/塔)は従来通り手動で開始する
4. quickResolveの即時モーダルが探索から消えている
5. フラスコ経済が手動と完全一致（同一関数を通るため構造的に保証される）
6. **ユーザー再ゲート15分**で3問テスト: Q1考えた=ボス戦/編成、Q2欲をかいた=捕獲パネルと深度、Q3気持ちいい=揺れ→カチッ

## 6. ゲーム全体での位置づけ

```
拠点(相棒と身支度) → ノード探索(どこへ潜るか) → オート観戦バトル(観る)
   → 捕獲チャンス(選ぶ・欲をかく) → 揺れ→カチッ(高揚) → 図鑑が埋まる(収集)
   → ボス戦だけ手動(考える) → 記章 → 次の世界
```
「気軽に遊べるけど収集はしたくなる」の絶妙ラインは、**操作を捨てて儀式を磨く**ことで作る。
