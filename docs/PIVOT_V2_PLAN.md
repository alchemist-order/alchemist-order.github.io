## 1. A層: 移動廃止 → 「ノード探索」への置換設計

### 1.1 診断（なぜ移動を疑うのが正しいか）
- 歩行のカクつきは実装品質の問題だが、**根本はタイル歩行がスマホブラウザと相性最悪**（誤タップ・視点の狭さ・片手操作不能）
- コアループ（遭遇→捕獲→錬成→塔）に歩行は**一切必須でない**。歩行が担うのは「世界の雰囲気」と「発見の間」だけ
- arkhomuは完全メニュー駆動で「午前3時」を作った。市場実証あり

### 1.2 置換設計: ノード探索（歩かない探索）
```
ワールドマップ(一枚絵) → エリア選択(森の入口/奥地/マグマ湖畔…) → 「探索する」タップ
  → イベントフィード形式で結果が流れる:
     「草むらで エンベリオが 飛び出した！」→ バトルへ
     「熾火の欠片 を拾った」
     「宝箱を見つけた！」
     「ヌシの気配がする……挑む/引き返す」
  → 3〜5イベントで1探索終了。スタミナ無し・連続探索可
```
- **既存資産を全部再利用**: エリア＝既存マップのzones、遭遇プール・ヌシ・宝箱・レア枠はそのまま。マップ俯瞰絵(bg/map/*.png 9枚、現在未使用)が**ワールド/エリア選択画面の主役**としてついに活きる
- 「欲をかく」設計を移植: 探索を続けるほど深部イベント(レア/ヌシ/極素材)の重みが上がるが、手持ちHPは回復しない → **「もう1回潜るか、帰るか」**が毎回の意思決定（Package Dの精神をUIだけ変えて継承）
- NPC/ストーリーは「拠点画面の会話リスト」と「探索イベント」に統合。F4の序盤テキストは会話イベントとして温存

### 1.3 移行手順 【2026-07-15改訂: 歩行廃止を確定】
**ユーザー30分ゲートの結果「序盤で嫌気」＝作者本人ですら耐えない一次データが出たため、A/B並存案は破棄。即時置換を確定。**
1. O13: ノード探索へ全面置換（詳細= [SPEC_NODE_EXPLORATION.md](SPEC_NODE_EXPLORATION.md)）。Field.tsx/タイル歩行は廃止
2. 歩行実装への投資は全停止（Codexはキャラ・イベントカード用の絵に全振り）
3. 置換後にユーザー再ゲート(15分版)→合格でソフトローンチ

## 2. B層: やりこみ最強の設計（縦の深さ＝継承世代）

既定路線（素材錬成24種/派遣/週塔/厳選/変異種/実績）に、**arkhomuの核心「ビルド語彙」**を1つ足して完成させる:

### 継承世代システム（錬成の縦掘り）
- 錬成のたびに「世代」が上がり、**継承スロット**が増える(G1=1枠→G5=4枠)
- 素材側の技・特性・才能ボーナスを選んで継承 → 「理想の1体」までの道のりが長く、毎回意思決定がある
- 週塔・対ジム再戦で「自分のビルド」を試す場が既にある
- 効果: 図鑑コンプ(横)×継承ビルド(縦)×週塔(競争)の3軸が揃い、**遊び尽くせない構造**になる
- 実装規模: 中（fuseResultの拡張＋UI）。O12素材錬成の次

### どっぷり接着剤（軽量・全て設計済み）
ログボ小包／派遣の帰還待ち／週替わり塔／デイリー。※スタミナは導入しない(BENCHMARK §7.5の結論維持: 遊びたい人を止めない)

## 3. C層: 課金の現実（ここだけ階段を飛ばせない）

### 3.1 「コインを売る」に必要なもの（全部揃うまで売れない）
1. **サーバー権威**: localStorageの残高は誰でも書き換えられる。購入情報はサーバー管理必須＝**F5(Supabase)+アカウントが前提**
2. **決済**: Stripe等の導入・本人確認・特商法表記・返金対応
3. **法務**: 有償通貨は**資金決済法の前払式支払手段**に該当。未使用残高の管理義務、基準日残高5,000万円超で供託義務等。個人が背負うには重い制度
4. **経済設計**: 売る前に「コインで何を買うと嬉しいか」が要る（現状は売り物が無い）

### 3.2 現実的な収益化の階段（この順で）
| 段 | 手段 | 前提 | 法務負担 |
|---|---|---|---|
| 1 | 投げ銭(OFUSE/Ko-fi) | なし・今すぐ | ほぼ無し |
| 2 | **買い切りアンロック**（例: コスメパック/追加セーブ枠/サポーターバッジ。itch.io経由なら決済も店側） | 少数の熱心なファン | 軽（前払式に該当しない一回性） |
| 3 | Steam買い切り版 | M3ゲート通過 | Steamが決済・返金を代行 |
| 4 | **コイン(有償通貨)** | F5稼働＋MAU数千＋2〜3の売上実績 | 重（資金決済法・要専門家確認） |
- 転換率の相場は1〜3%。**プレイヤー1,000人でも課金者は10〜30人**。段4はその規模になってから初めて意味を持つ
- **無料でどっぷり遊べる版が先。課金は「深く遊んだ人がお礼を払える出口」から作る**（キャラ販売ガチャ型は最後の最後）

### 3.3 いま決めること
- V2設計は**将来の課金を邪魔しない形**にしておく: 通貨は「ゲル(無償)」と将来の「錬金石(有償想定)」を最初から別スロットで設計。ただし**販売は実装しない**
- 「魅力的なキャラで売る」はまず**無料の看板**として使う（LP/X/図鑑）。売り物にするのはファンがついてから

## 4. D層: キャラ全面化（即実行・低コスト）

1. **ホーム画面の主役交代**: メニューリストの上に「相棒が大きく立つ」（立ち絵 or スプライト拡大＋タップで鳴く/喜ぶ）
2. **図鑑のリッチ化**: 1体詳細を全画面カード化（大きな絵・フレーバー・入手法・変異種トグル）＝シェアしたくなる図鑑
3. **勝利・捕獲・錬成の演出でキャラを寄せる**（既存GetOverlay/ShareCardの流用拡張）
4. LP/X運用もキャラ推し投稿を主軸に（PROMO ④図鑑紹介の比重UP）
