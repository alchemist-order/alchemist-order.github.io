## 1. 設計原則

1. **タイルマップは作らない**。ステージ=データ定義のみ(背景は既存bg/map・bg/battleの流用割当)
2. **プールは手書きしない**。タイプ重み×段階からの**決定的自動割当**(§3)＋カバレッジ検査で293種全員を保証
3. **プレビューが収集の入口**: 選択時に「出現幻獣(未発見はシルエット???)」「ドロップ」「推奨Lv」を見せる——未発見シルエットこそ次の目的地を決めさせる装置
4. 解放は同一世界内で直列(前ステージ踏破で次)、世界間は既存の記章ゲート維持。深域3つは記章数ゲート

## 3. プール自動割当（実装ルール・手書き禁止）

```
src/game/stages.ts:
export interface StageDef { id; worldId; name; desc; band:[min,max]; typeWeights:Record<string,number>;
  tiers:number[]; poolSize:number; rare?:{tiers:number[]; chance:number};
  nushiId?; chestCount?; bossTrainerId?; unlock:{ prev?:string; badges?:number; badge?:string };
  dropTable:{key:string; rate:number}[]; bg:string; extra?:string[] }

buildStagePool(stage): 
  1. candidates = DEX.filter(d => d.role!=='legendary' && stage.tiers.includes(d.stage) && stage.typeWeights[d.type])
  2. 決定的順序: cyrb53(d.id + '|' + stage.id) 昇順でソート（Math.random禁止・毎回同一）
  3. タイプごとに typeWeights の比率で poolSize(既定10) 件をラウンドロビン選出
  4. stage.extra(明示追加id)を先頭にマージ
  rarePool(stage): 同様に rare.tiers から4件
```
- **カバレッジ保証**: `scripts/check_stage_coverage.mjs`(新規)——全293種が「いずれかのステージのpool/rarePool/extraに含まれる」ことを検査し、漏れをid列挙で**exit 1**。`npm run build` の check_rng の隣に組み込む。漏れが出たら該当タイプのステージに `extra` で明示追加して解消
- 遭遇生成: 既存 makeWild に pool を渡す現行フローのまま(Lv帯はstage.band)。チェーン/今日の幻獣(O15)の補正もそのまま効く

## 5. ステージ選択UI（プレビュー＝収集カタログ）

Explore.tsx を「世界タブ → ステージグリッド → ステージ詳細パネル」に再構成:
1. **ステージカード**(グリッド・スマホ2列): 名前/推奨Lv帯/状態アイコン(✅踏破・🐾ヌシ・🎁宝箱残・⚔ボス)/未解放は🔒+解放条件
2. **詳細パネル**(カードタップで展開 or 下部シート):
   - 出現幻獣: buildStagePool の全件をスプライト小で表示。**seen済=スプライト+名前 / 未発見=シルエット(brightness(0))+「？？？」**。レア枠は金枠+★
   - ドロップ: ItemIcon列+確率語(よく出る/ときどき/まれに)
   - [🧭 探索する](主CTA) [🚀 おまかせ遠征](**踏破済みのみ表示**=O15 §6と同一ゲート)
3. 踏破定義: そのステージで1セッション3イベント以上消化して帰還 or ヌシ/ボス撃破 → `progress.clearedStages[]` に記録(GameState拡張・旧セーブ補完)
4. ボスステージは従来どおり手動バトル固定・戦前台詞維持
