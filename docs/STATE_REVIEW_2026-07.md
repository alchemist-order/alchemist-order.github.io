## 2. 足りないもの（優先度順・率直に）

### P0-1: 実プレイヤーの目（最大の欠落）
30分ゲート以降、**改修後の姿を通しで遊んだ人間がゼロ**のまま機能が積み上がっている。
「クイック決着違和感」の発見はゲートの成果だったが、その修正版(V3)も未ゲート。
**機能はもう足りている。足りないのは検証。** → 機能追加を凍結し15分ゲートへ。

### P0-2: 重さ＝ブラウザゲーム失格級の問題（今回の計測で発覚）
- **dist合計105MB**（Webゲームの経験則は30MB以下）
- 探索画面が `explore-backdrops` で **bg/map全9枚=19MBを開幕プリロード**（装飾のための隠しdiv）
- ui/のアイコン類が512px級のまま(100-200KB×多数)、logo 812KB、スプライト31MB(遅延ロードだが1枚100KB級)
- 「URLを踏んで3秒で始まる」がUSPなのに、**スマホ回線では3秒で始まらない**
→ 対策(§3-②)は機械的で効果が大きい。ローンチ前必須。

### P1-1: セーブの信頼性
localStorage単独。ブラウザの「サイトデータ削除」で全消滅・手動書き出しのみ。
長く遊ばせるゲームで**資産が消える恐怖への備えが無い**のは致命傷になり得る。

### P1-2: 最初の3分（FTUE）の未設計
歩行版の導入(F4の物語)を作り込んだが、新UI版で「開始→相棒→**最初に何を押すか**」の誘導が未検証。母・師・ミルカの台詞が拠点に移植されているかも未確認。Webゲーム経験則「最初の3分で迷わせない・死なせない」。

### P1-3: 閉じている間に何も起きない
再訪動機がログボだけ。放置系の核「戻ったら何かある」(派遣調査=O12 §7・設計済み未実装)が空席。

### P2（ローンチ後でよい）

## 4. ひとこと

機能の総量はもう「売り物」の水準にある。足りないのは **軽さ**（105MB問題）と **他人の目**（ゲート・ローンチ）だけ。
この2つを飛ばして機能を足し続けると、「誰も遊んでいない豪華なゲーム」が完成してしまう。次の1手は開発ではなく計量と検証。
