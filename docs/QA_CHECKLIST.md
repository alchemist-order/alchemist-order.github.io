# QA回帰チェックリスト（S2・Sonnetランブック）

策定: 2026-07 / 用途: リリース(mainへのpush)毎に流す回帰確認。
対象フロー: **新規開始→森→塔→錬成→セーブ入出力**。
実行方法: `mcp__Claude_Preview__*` でヘッドレス操作(このプロジェクトの標準手順)。
既知の環境クセ(手順内で対処済み):
- `preview_screenshot` は不安定→DOM/eval中心で検証する
- 初回マウント直後は `.viewport` の `clientWidth` が0になることがある→`preview_resize`で明示リサイズしてから座標計算する
- クリック直後に同期で`document.body.innerText`を読むとReactの再描画前の値を拾う→`await`で最低100ms待つ
- タイトル画面のボタンラベルはセーブ有無で変わる(`はじめる`/`さいしょから`/`つづきから`)

## 手順と合格基準

### 1. 新規開始
1. `localStorage.removeItem('alchemist-order-save-v1')` → リロード
2. タイトルの開始ボタン→プロローグ9ページが順に表示され`目を覚ます`で終了
3. ログインボーナス等の初回ダイアログを閉じる→わが家2階(home2f)のベッド付近で開始
4. 母(home,3,2)に話しかけ「傷薬を三つ」を受け取る→`items.heal`が+3
5. 師ガレン(mentor_house)に話しかけ御三家モーダル→1体選択→`GetMonsterOverlay`→師の激励台詞
6. **合格基準**: `game.collection.length===1`、選んだ種がフィールドで先頭に立つ、コンソールエラー無し

### 2. 森（フィールド/エンカウント/ヌシ/ゲート）
1. 中央広場の転送門(rapis,17,20)→世界選択モーダル→緑霧の森へワープ
2. 高草(`G`)を数回踏み、通常の野生エンカウントが発生することを確認(`ENCOUNTER_RATE`)
3. 区画別エンカウント: 草地F(8-14,9-12)相当のマスで、ベース設定(Lv4-8)より高いLv帯(11-14)が出ることを確認
4. ヌシ(17,29)に接触→前口上→固定個体(マンドラゴLv13★★)とのバトル開始→勝敗いずれかで終了
5. **合格基準**: 通常戦闘/ヌシ戦闘とも正常に開始・終了しフィールドに戻る、コンソールエラー無し

### 3. 塔（試練の塔・シード付きラン）
1. 転送門モーダルの「🗼試練の塔」→挑戦→パーティ満タンで開始
2. 1階クリア→2階へ自動遷移(Lv上昇を確認: 4+floor*2)
3. 敗北 or 撤退で終了→自己ベスト記録のダイアログ
4. **合格基準**: 階層が正しく進行し種/Lvがシードに応じて決定論的(同一ビルドなら再実行しても分布が安定)、コンソールエラー無し

### 4. 錬成（融合）
1. 手持ちを2体以上・所持金300以上にする
2. 錬成師ミルカ(mentor_house,2,5)に話しかけ→錬成モーダル(2体未満なら導入台詞のみ)
3. ベース/素材を選択→実行→結果個体(`GetMonsterOverlay`)+ログへ反映
4. **合格基準**: `collection`から素材2体が消え結果1体が増える、`money -= 300`、コンソールエラー無し

### 5. セーブ入出力
1. 設定(⚙)→「セーブを書き出す」→コードが textarea に表示される
2. `localStorage.removeItem` でセーブを消す(=初期化相当)
3. 書き出したコードを「読み込み」欄に貼り付けて実行
4. **合格基準**: `importSave`成功時は元の`collection`/`money`/`pos`等が完全一致で復元、失敗系(不正コード)は「コードが正しくないようだ」を表示してクラッシュしない
