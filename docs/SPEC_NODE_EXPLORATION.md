## 2. ワールドとノード（既存zonesの読み替え）

各世界＝ノード列。ノードは奥へ一方向に解放（前ノードの探索クリアで次が開く）。

| 世界 | ノード（エンカ帯は既存zones/poolをそのまま割当） |
|---|---|
| 緑霧の森 | 森の入口 → 深い森 → 霧の最深部 → ⚔守護者シルヴァ |
| 潮鳴りの海 | 潮騒の道 → 磯の岩場 → 港の外れ → ⚔守護者マレア |
| 紅蓮の火山郷 | 溶岩回廊 → マグマ湖畔(ヌシ) → 火口への道 → ⚔守護者イグナート |

- ノード選択画面の背景に `bg/map/<元マップid>.png`（9枚全部活きる）
- ジムノードは世界の最終ノード。**リーダー戦前の会話・敗北時台詞は既存TRAINERSを流用**
- 第4世界以降もノード定義を足すだけ＝**世界追加の実装コストが激減する**(タイルマップ制作が不要になる)

## 3. 探索セッション（コアループ）

```
「探索する」タップ
 → イベントを1つずつ提示（テンポ: タップ送り、オート送り設定可）
 → 3イベント消化で「区切り」: [ さらに進む(深度+1) / 帰還する ]
 → 深度が上がるほど: 遭遇Lv+1〜2 / レア枠・良素材・宝箱の重み増 / ヌシ遭遇解禁
 → 帰還で戦利品確定。**手持ちHPは探索を通して持続**(宿まで回復なし)
 → 全滅: 拠点に戻され、その探索で得た戦利品の半分を失う
```
**設計意図**: Package Dの「欲をかく」意思決定（進むか帰るか）を歩行なしで再現する。1探索=1〜3分でスキマ時間に収まる。スタミナは導入しない（BENCHMARK §7.5維持・何回でも潜れる）。

### イベントテーブル（ノードごとに重み付け）
| イベント | 内容 | 既存資産 |
|---|---|---|
| 遭遇(主) | 野生バトルへ | encounter/zones/pool・レア枠そのまま |
| 素材発見 | 素材アイテム入手(O12連動。実装前は金/傷薬) | SPEC_ITEM_ALCHEMY §2 |
| 宝箱 | 既存chestsの中身をイベント化(1ノード累計上限=旧chest数) | chests |
| ヌシの気配 | 「挑む/引き返す」→固定個体戦 | nushi(3体)そのまま |
| ルーン盤 | 起動するとショートカット解放=次回からそのノードへ直行可 | switches/gates読替 |
| 人物・フレーバー | 旅人の会話/景色の描写(1〜2行) | signs/villager台詞を再利用 |

- 乱数はfieldストリーム(非シード)のまま。イベント抽選は「同イベント連続を抑制」する重み減衰を入れる
- **演出**: イベントカードにはノード背景+関連画像(幻獣シルエット/宝箱/素材アイコン)。テキストだけにしない

## 6. バランス初期値（Sonnet検証対象）

- 1探索=3イベント+深度選択。ジム解放まで: 森=のべ3〜4探索(約10分)を目安
- 遭遇率(イベント配分): 遭遇55% / 素材20% / フレーバー15% / 宝箱・特殊10%
- 深度ボーナス: 深度+1ごとに レア枠+5pt・敵Lv+1(上限=旧zones最深部Lv)
- 受け入れ: sim不要(乱数テーブルの静的検証で可)。**S2 QA手順書を本specの動線で全面改訂**
